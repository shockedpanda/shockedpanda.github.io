Our services & tools

Everything we offer – from audits to fully built tools – is available on request. You only pay for what you use, and we open‑source stripped‑down versions for DIY teams.

Compliance Automation Audit

We map your current KYC/AML workflows, identify repetitive high‑volume tasks, and deliver a prioritised automation roadmap. You receive a concrete report, not just advice. The audit is a one‑time engagement that serves as the blueprint for any pilot or full deployment.

Pilot Build

We adapt OnboardAI and PolicyPulse to your exact documents and deploy them on your infrastructure – self‑hosted or fully guided setup. The pilot typically runs for 4‑6 weeks and demonstrates tangible time savings. No vendor lock‑in: the code runs on your hardware, and you keep full ownership of any customisations.

Ongoing Support

A monthly retainer that covers updates, new features, and maintenance as your requirements evolve. Your data never leaves your control. Support also includes access to our private repository of compliance automation modules and priority email/chat assistance.

Local LLMs • Docker • No Cloud

All our tools run on Llama 3.1, OCR, and containerised deployment (Docker). You stay in control – your data never leaves your infrastructure. We provide pre‑built container images, orchestration scripts, and documentation for air‑gapped environments.

On‑demand tools (build on request)

Any of the following can be built for your organisation. After delivery, we release a stripped‑down open‑source version on GitHub for the community.

OnboardAI – Automated KYC Onboarding

What it looks like: A folder watcher + web dashboard. Drop client packs (PDFs, images, Word files) into a monitored folder. The tool extracts identity data, runs sanctions/PEP checks, and generates a structured Word report. Example output: A 5‑page report with risk score, extracted fields, and a compliance checklist. How it works: Uses local Llama 3.1 via Ollama, Tesseract OCR, and a rule‑based completeness validator. No data ever leaves your server.

PolicyPulse – AML Policy Update Engine

Current POC: Runs in the background – you drop your AML policy plus a new regulatory notice into a folder. The tool performs a section‑by‑section comparison, flags changes, and outputs a change log table with citations and a “reviewer checklist” (Word or PDF). Example output: A report that guides compliance officers through each impacted section, showing old vs. new wording and suggested actions. How it works: Uses a local vector database (Chroma) to compare semantic meaning, not just text diffs. Learns from past regulatory updates to improve accuracy. Optional UI: For clients who prefer an interactive side‑by‑side view, we can build a simple two‑panel interface on request.

KYC Document Certification Validator

What it looks like: A bulk upload screen where you drop certified copies (e.g., passports, utility bills). The tool scans each document for proper certification wording (“I certify this is a true copy”), date within 3 months, and certifier’s credentials. Example output: A colour‑coded spreadsheet: green = compliant, yellow = warning (expiring soon), red = reject. How it works: Combines OCR with a custom rule engine that understands 15+ certification formats used by different jurisdictions.

STR/SAR Drafting Assistant

What it looks like: A dashboard showing flagged transactions with risk scores. Clicking a flag opens a draft STR narrative pre‑filled with transaction details, unusual pattern descriptions, and regulatory citations. Example output: A complete STR ready for MLRO review – saves 2 hours per report. How it works: Uses anomaly detection on transaction data (amount, frequency, counterparty) and an LLM to generate narrative in your jurisdiction’s required format (e.g., FINTRAC, FinCEN, MAS).

Compliance Librarian

What it looks like: A chat interface (like ChatGPT) connected to your secure document store. You ask: “Show me all expired CDD for client Acme Corp”. The tool returns a list of documents with expiration dates and can generate a renewal request email. Example output: “3 documents expired: Passport (expired 2025-01-10), Utility bill (expired 2025-02-01).” How it works: Uses a local embedding model to index your documents; queries are converted to vector search. All data stays on‑premise.

Ongoing Monitoring Engine

What it looks like: A scheduled job that runs nightly. It re‑screens your entire client database against updated sanctions lists and scrapes adverse media (news, watchdogs). Example output: A daily email summary: “3 clients newly matched to OFAC SDN list – review required.” How it works: Uses fuzzy matching (Levenshtein, phonetic) to catch name variations. Only reports changes, not full re‑screens, to reduce noise.

Corporate Documents Librarian

What it looks like: Specialised version of Compliance Librarian focused on corporate entities. Upload incorporation certificates, registers of directors, shareholder lists. Example output: A “corporate health dashboard” showing which entities have missing documents, expired certificates, or upcoming filing deadlines. How it works: Adds extra parsers for corporate document types (e.g., extracts UBO percentage from shareholder registers).

Board/MI Report Compiler

What it looks like: A report generation wizard. Select date range, KPIs (e.g., number of alerts, average review time, sanctions hits). Example output: A 10‑page board pack with charts, tables, and executive summary, in Word or PDF. How it works: Pulls data from your existing compliance systems (via API or database connectors) and uses a template engine to produce a consistent, branded report.

Regulatory Change Impact Assessor

What it looks like: Upload a new regulation (PDF or text). The tool maps each article to your internal controls (policies, procedures, systems). Example output: A matrix showing “Control 4.2 – Client Risk Rating” is impacted by Article 7 of the new regulation; suggested action: “Update risk scoring model”. How it works: Uses an LLM to extract obligations and a knowledge graph of your controls. Output includes a prioritised remediation plan.
